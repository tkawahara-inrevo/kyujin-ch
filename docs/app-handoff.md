## 1. プロジェクト概要

**サービス名**: 求人ちゃんねる (kyujin-ch.jp)
**運営**: 株式会社INREVO
**サービス内容**: 求職者向け求人媒体（toC）+ 採用支援サービス（toB）

### 現状の構成

- Webアプリ: Next.js 16 (App Router) + React 19 + TypeScript
- データベース: PostgreSQL (AWS RDS via Lightsail)
- インフラ: AWS Lightsail (Next.js: 13.159.236.233, WordPress: 18.179.136.107)
- ドメイン: kyujin-ch.jp（メイン、Next.js）、wp.kyujin-ch.jp（WordPress）
- 認証: NextAuth 5 (Cookie ベース)
- 主要機能:
  - 求人検索・詳細・応募
  - メッセージ機能
  - マイページ・履歴書作成（AI支援）
  - 企業ダッシュボード・求人作成・応募者管理
  - 管理者画面（求人審査・コラム管理 等）
  - Focus（企業特集メディア）
  - toC/toBコラム

### サーバー費用（参考）

- 月額: 約 ¥6,000〜¥13,000（Lightsail + RDS + S3 + SES + ドメイン）

## 2. アプリ化の決定事項

### 方式

**Swift（iOS）+ Kotlin（Android）の完全ネイティブ実装**

#### 検討経緯
1. 当初 Capacitor / WebView ラップを検討
2. → 「アプリ独自の挙動」要望と品質重視で却下
3. → Flutter / React Native / Native の3択で比較
4. → 業界傾向（Indeed/リクナビ/マイナビ/DODA等 = ほぼNative）と品質高め希望から、Swift+Kotlinに決定
5. ビズリーチがRN→Nativeに書き直した事例も参考

### 体制

- エンジニア: ユーザー（1名）+ Claude（AI）
- デザイナー: 社内（1名）
- 外注なし

### 初期スコープ（MVP）

- 求人閲覧
- 応募
- メッセージ

→ それ以外（Focus、企業側ダッシュボード等）は MVP には含めず、後続アップデートで追加

### アプリ独自機能

- Push通知（必須）
- Biometric認証 / Face ID・指紋（必須）
- オフライン閲覧（必須）
- その他（依頼者にヒアリング中）

### 工期目安

- 最短10ヶ月、現実14ヶ月（2人体制）

### コスト

- Apple Developer Program: $99/年（約¥15,000）
- Google Play Developer: $25 初回のみ（約¥4,000）
- その他（Mac、iPhone実機、デザインツール等）: 既存リソース活用想定
- **追加金銭コストは¥1.9万のみ**
