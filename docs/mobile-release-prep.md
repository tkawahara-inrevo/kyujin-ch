## [1] Google Play Developer 登録

### 料金
- **$25 (1回払い・永続)**
- クレカ必要 (VISA/MasterCard/AMEX)

### URL
https://play.google.com/console → 「Get started」

### 個人 vs 法人

| 項目 | 個人 | 法人 |
|---|---|---|
| 登録手間 | 即日〜数日 | D-U-N-S 番号取得が必要 (1〜2週間) |
| 公開される名前 | 個人名 (例: Takahiro Kawahara) | 法人名 (例: 株式会社インレボ) |
| ストア表示の信頼感 | △ | ◎ |
| 後から変更 | **個人→法人移行可** (一度移行すると戻せない) | - |

**推奨**: まず個人で始めて、本リリース前に法人移行。

### 必要なもの
- Google アカウント
- クレカ
- 身分証 (パスポート or 運転免許の写真)
- 連絡先メール・電話番号
- 公開する開発者名 (本名 or 任意)
- 公開連絡先メアド (サポート用)

### 法人で登録する場合 追加で必要
- **D-U-N-S 番号** (https://www.dnb.com/jp-ja から無料申請、取得2週間)
- 会社住所・電話番号
- 会社ウェブサイト URL

### 承認時間
- 個人: 即時〜数日
- 法人: D-U-N-S 確認のため 1〜2 週間

## [2] 署名鍵 (Keystore) について

**重要**: 一度作ったら**絶対に紛失しない**。署名鍵を失うとアプリのアップデートができなくなる。

### 私が作成する内容
- `kyujinch-release.keystore` ファイル
- パスワード (高強度ランダム)
- key alias と key パスワード

### 保管方法 (takarさんの作業)
- ファイル: 1Password 等のパスワードマネージャに添付
- パスワード: 同じく 1Password に保管
- バックアップ: 別場所にも保管 (Google Drive 暗号化 + 紙メモ等)

### Google Play App Signing
- 推奨: Play App Signing 有効化
  - Google が上書き署名キーを管理
  - 万が一の紛失時に再発行可能
  - 初回アップロード時に「上書き署名キーをアップロード」する流れ

## [3-4] アプリ情報 (Play Console)

### 必須項目

| 項目 | 例 / 注意 |
|---|---|
| アプリ名 | 求人ちゃんねる |
| 簡単な説明 (80文字) | 求人検索・応募・メッセージのオールインワンアプリ |
| 詳細な説明 (4000文字) | サービス全体の説明 |
| アプリのカテゴリ | ビジネス or 仕事 |
| **連絡先メール** | 公開される |
| **プライバシーポリシー URL** | 必須・URL 必須 |
| アプリのアイコン | 512×512 PNG (高解像度) |
| **特集グラフィック** | 1024×500 PNG |
| **スクリーンショット** | 最低2枚、推奨8枚 (各 OS で異なる解像度) |
| 対象年齢 | 「すべて」or「12+」等 |
| **コンテンツレーティング** | アンケート回答で自動算出 |
| **データセーフティ** | 収集するデータ・用途を申告 |
| **広告の有無** | あり/なし |

### プライバシーポリシー

- **URL 必須**: https://kyujin-ch.jp/privacy 等
- 既存の Web 版プライバシーポリシーで OK (モバイルアプリのデータ収集を追記)
- 追記すべき内容:
  - FCM トークンの保存
  - 端末識別子 (Android ID) の保存
  - 任意の生体認証 (端末内で完結する旨)

### データセーフティ (重要)

申告内容:
- **収集データ**: 氏名・メアド・住所・電話番号 (任意) / 求人閲覧履歴 / 端末識別子
- **共有相手**: 応募時に企業に開示
- **暗号化**: HTTPS 通信中
- **削除リクエスト**: アプリ内退会で可

## [5-6] AAB アップロード + 内部テスト

### Android App Bundle (AAB)

- APK ではなく **AAB** が推奨形式 (Google Play 必須)
- ビルドコマンド: `./gradlew bundleRelease`
- ファイルサイズ削減 + デバイス別最適化

### 内部テスト トラック

- **最大 100 人** までテスター招待可能
- メアド or Google グループで招待
- テスター: 招待リンクから opt-in → Play ストアでインストール
- **審査不要**・即配信

### テスター招待の流れ
1. Play Console → 「内部テスト」
2. 「テスター」タブ → 「メーリングリストを作成」
3. メアド追加 (1行1メアド or CSV)
4. リリースを作成 → AAB アップロード → 「公開」
5. 「テスター」タブで「リンクをコピー」
6. テスターに共有

### テスターの体験
1. 招待メールリンクをタップ
2. Google アカウントで Opt-in (1回だけ)
3. 「**Play ストアで開く**」リンクが表示される
4. **通常のアプリと同じインストール手順**
5. アップデートも自動

## [7] テスト実施

### バージョン管理
- `versionCode` (整数): アップロードごとに +1
- `versionName` (文字列): "0.1.0", "0.2.0" 等

### テスト依頼の出し方
- 招待リンク (Play Console から取得) を共有
- 確認してほしい機能をテストケースとして列挙
- バグ報告先 (Slack? Email?) を決めておく
