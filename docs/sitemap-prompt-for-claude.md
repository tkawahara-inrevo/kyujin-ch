# 【claude.ai に渡す用】サイトマップ → 遷移図 生成依頼テンプレート

このファイル全体（このメッセージ全文）を **claude.ai (https://claude.ai)** にコピペするだけで、サイトマップから遷移図が生成されます。

---

## 依頼内容

求人媒体サービス「**求人ちゃんねる**」(https://kyujin-ch.jp) の画面一覧と遷移情報を渡します。以下を作成してください:

### 1. Mermaid 形式の遷移図（複数）

ユーザータイプ別に分けて、見やすい遷移図を **Mermaid flowchart** で作成してください:

- **① 求職者導線（未ログイン〜応募完了まで）**: ホーム → 求人検索 → 求人詳細 → 応募 → メッセージ までの自然な流れ
- **② 求職者 ログイン後の画面構成（マイページ系）**: マイページから各機能への遷移
- **③ 企業導線（ログイン〜応募者対応まで）**: 企業ログイン → ダッシュボード → 求人投稿・応募者管理
- **④ 管理者導線**: 管理者ログイン後の各機能へのアクセス
- **⑤ Android アプリ導線**: アプリ起動〜各機能の遷移

各図は **markdown コードブロック (```mermaid) で出力**して、claude.ai 上で即プレビューできるようにしてください。

### 2. 短い解説テキスト

各図の上に、その導線の意図や注意点を 2〜3 行で説明してください（例: 「求職者の主要動線。応募完了後は応募一覧に遷移する設計」）。

### 3. 注意点

- 図が大きくなりすぎる場合は **無理に1枚にまとめず分割** OK
- ユーザータイプ横断の遷移（求職者 ↔ 企業のメッセージ等）は **重要なものだけ点線で**
- 図中の画面名は **TSV の「画面名」列をそのまま使う**（URL は省略可）
- 認証必要な画面は 🔒 アイコンを付けると分かりやすい

## 列の意味

- **区分**: Web / アプリ
- **階層**: L1 (トップ) → L4 (深い操作画面)、または 求職/企業/管理/認証/タブ/詳細/マイページ/特殊
- **画面名**: 画面の名称
- **URL**: ルートパス（アプリは Composable 名）
- **対象**: 主な利用者
- **認証**: 「不要」or「要ログイン」
- **主な遷移元**: この画面に来るときの典型的なルート
- **主な遷移先**: この画面から次にどこに行くか
- **説明**: 画面の役割
