# 管理者マニュアル

求人ちゃんねる運営側の管理画面の使い方。

## アクセス

- URL: https://kyujin-ch.jp/admin/login
- ユーザー名 or メアド + パスワード

## ロール

| ロール | 権限 |
|---|---|
| **SUPER_ADMIN** | 全機能フルアクセス |
| **ADMIN** | `adminPermissions` で個別に権限設定（求人・企業・求職者・コラム等を個別ON/OFF） |
| **SEO_EDITOR** | コラム編集のみ |

ADMIN ユーザーは権限のない機能には URL 直アクセスしても /admin にリダイレクトされる。

## メニュー（左サイドバー）

| メニュー | 内容 | 権限 |
|---|---|---|
| ダッシュボード | 各種指標一覧 | dashboard |
| 企業一覧 | 企業管理 | companies |
| 求職者一覧 | 求職者管理 | jobseekers |
| 求人一覧 | 求人審査 | jobs |
| コラム管理 | 求職者向けコラム編集 | columns |
| toBコラム管理 | 企業向けコラム編集 | columns |
| Focus 記事管理 | 特集記事編集 | focus |
| 請求単価管理 | 請求単価設定 | billing |
| 請求一覧 | 月次請求書管理 | invoices |
| 無効申請 | スパム的応募の対応 | invalidRequests |
| 問い合わせ | お問い合わせ対応 | inquiries |
| 分析 | 各種統計 | analytics |

## 求人審査

### 一覧

- メニュー「**求人一覧**」
- フィルタ:
  - ステータス（DRAFT / 審査中 / 公開中 / 要修正 / 取り下げ済）
  - 企業
  - 期間
- 「**審査待ち**」ステータスのみフィルタすると、対応すべき求人だけ表示

### 個別審査

- 一覧から求人をクリック → `/admin/jobs/<id>` で詳細表示
- 左ペイン: 求人内容（求職者から見える状態）
- 右ペイン: 審査アクション

#### 承認

- 右ペイン「**承認して公開**」ボタン
- ステータス → PUBLISHED、即時公開
- ⚠️ 「下書き」「取り下げ済」「差し戻し済」の求人は承認不可（仕様）

#### 差し戻し

- 右ペイン「**差し戻し**」ボタン
- セクションごとに修正コメントを入力
- 入力済みコメントは企業側の編集画面に表示される
- ステータス → RETURNED

#### 公開中の求人の差し戻し（公開停止）

- 公開中の求人を差し戻すと**非公開** + RETURNED に
- 警告メッセージが表示される

### 審査履歴

- 上部に「**審査履歴 (N件)**」のアコーディオン
- 過去のステータス変更を時系列で表示
- 企業アクション（作成・取り下げ・再申請）と管理者アクション（承認・差し戻し）両方表示

## 企業管理

### 一覧

- メニュー「**企業一覧**」
- 企業名検索・新規企業追加
- 各企業の応募数・公開求人数等を表示

### 個別企業

- 詳細ページで企業情報を表示・編集
- **代理ログイン**機能: 企業画面に管理者として入って動作確認可

### 新規企業追加

- 企業情報を入力 → 代表者（COMPANY ユーザー）情報を入力
- 自動でメール送信（パスワード設定リンク）

## 求職者管理

### 一覧

- メニュー「**求職者一覧**」
- メアド検索・期間絞り込み

### 個別求職者

- 詳細ページでプロフィール・応募履歴を表示
- アカウント無効化・強制削除可能（ストア審査要件）

## コラム管理

### 求職者向け（columns）

- メニュー「**コラム管理**」
- 一覧・新規作成・編集
- tiptap エディタでリッチテキスト編集
- カテゴリ・サムネイル設定可
- 公開設定で /column に反映

### 企業向け（biz-columns）

- メニュー「**toBコラム管理**」
- WordPress に投稿
- Next 側で /biz-column としてレンダリング

## Focus 記事

- メニュー「**Focus 記事管理**」
- 特集記事の編集
- /focus 配下に表示

## 請求管理

### 請求単価

- メニュー「**請求単価管理**」
- 企業ごとの応募1件あたりの単価を設定
- カテゴリ・雇用形態ごとに細分化可

### 請求書

- メニュー「**請求一覧**」
- 月次で自動集計
- PDF ダウンロード・Excel ダウンロード
- 入金処理（「**入金済み**」マーク）

## 無効申請

- メニュー「**無効申請**」
- スパム的な応募・テスト応募の管理
- 「無効」とマークすると請求対象から除外

## お問い合わせ対応

- メニュー「**問い合わせ**」
- カテゴリ別（求職者・企業・その他）に一覧
- ステータス: OPEN / 対応中 / クローズ
- 個別表示で返信・ステータス更新

## 通報対応

- 現状、専用UIなし。DB を直接見る or 将来 UI 追加予定
- 求職者の通報 (`/api/v1/reports`) は Report テーブルに保存される

## 分析

- メニュー「**分析**」
- 各種指標:
  - 新規登録者数（日次・月次）
  - 公開求人数
  - 応募数
  - PV
- グラフ表示

## サブアカウント管理（SUPER_ADMIN のみ）

- メニュー「**アカウント**」
- ADMIN / SEO_EDITOR の追加・削除
- ADMIN の `adminPermissions` を細かく設定

## トラブル対応

### 企業から「求人が公開されない」

1. 求人ステータスを確認
2. PENDING_REVIEW なら審査→承認
3. RETURNED なら差し戻し理由を企業に確認してもらう
4. WITHDRAWN なら企業が取り下げた状態。再申請を促す

### 求職者から「ログインできない」

1. 求職者一覧でメアド検索
2. `isActive: false` ならアカウント無効化済
3. パスワード忘れの場合はパスワードリセット案内

### 企業から「応募者にメッセージが届かない」

1. 応募者の `notificationsEnabled` 確認
2. 該当応募者からのブロックを確認

### Slack 通知が来ない

1. 環境変数 `SLACK_WEBHOOK_*` を確認
2. Slack 側で Webhook が無効化されてないか確認
